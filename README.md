# Кокпит — стартовый проект (проверка сервера)

Это маленькая, но настоящая версия: **вход/регистрация + общий список клиентов** из твоей базы Supabase. Задача — развернуть её и убедиться, что вся бесплатная связка работает: команда заходит под своими аккаунтами и видит общие данные. После этого вставим полный кокпит.

---

## 0. Что нужно один раз
- Аккаунт **GitHub** (github.com) — бесплатно.
- Аккаунт **Vercel** (vercel.com) — бесплатно, можно войти через GitHub.
- Node.js на компьютере — **необязательно** (нужен только если хочешь запускать локально; Vercel соберёт всё сам).
- Твой **anon public** ключ из Supabase: Project Settings → API → Project API keys → `anon` `public`. (Project URL уже подставлен: `https://acacnyjsvvbbsbspvnlf.supabase.co`.)

> Схему базы (таблицы) ты уже создаёшь по файлу SETUP-server.md (раздел «Схема базы»). Если ещё не сделал — сделай до деплоя.

## 1. Закинуть код в репозиторий (GitHub)

Выбери любой способ. Для не-разработчика проще **A** или **B**.

### Способ A — GitHub Desktop (мышкой, без команд)
1. Скачай и установи GitHub Desktop: desktop.github.com
2. Войди своим аккаунтом GitHub.
3. File → **New repository** → имя, например `cockpit`. Запомни папку, куда он его создаст.
4. Скопируй **содержимое** этой папки проекта (файлы `package.json`, `index.html`, `vite.config.js`, папку `src`, `.gitignore`, `.env.example`) в папку репозитория.
5. В GitHub Desktop появятся изменения → внизу слева напиши «init» → **Commit to main** → вверху **Publish repository** (сделай его Private).

### Способ B — Загрузка через сайт GitHub
1. github.com → **New repository** → имя `cockpit` → Create.
2. На странице репозитория → **uploading an existing file**.
3. Перетащи туда все файлы проекта (включая папку `src`).
4. Внизу **Commit changes**.

### Способ C — командная строка (если умеешь)
```bash
cd cockpit
git init
git add .
git commit -m "init"
git branch -M main
git remote add origin https://github.com/ТВОЙ_ЛОГИН/cockpit.git
git push -u origin main
```

> Важно: файл `.env` НЕ должен попадать в репозиторий (он в `.gitignore`). Реальный ключ кладём только в Vercel (шаг 2).

## 2. Развернуть на Vercel
1. vercel.com → войди через GitHub.
2. **Add New… → Project** → выбери свой репозиторий `cockpit` → Import.
3. Framework Preset определится сам как **Vite**. Ничего менять не надо.
4. Раскрой **Environment Variables** и добавь две:
   - `VITE_SUPABASE_URL` = `https://acacnyjsvvbbsbspvnlf.supabase.co`
   - `VITE_SUPABASE_ANON_KEY` = твой anon public ключ
5. **Deploy**. Через минуту получишь ссылку вида `https://cockpit-xxxx.vercel.app`.

> Каждый раз, когда обновишь код в GitHub, Vercel пересоберёт сайт автоматически.

---

## 3. Проверка, что всё работает
1. Открой ссылку Vercel → вкладка **Регистрация** → заведи себя (email + пароль).
2. Зайди. Добавь пару клиентов.
3. В Supabase → **Table Editor → profiles** → у своей строки поставь `role = owner`.
4. Пусть кто-то из команды откроет ту же ссылку и зарегистрируется — он увидит **тех же** клиентов. Это и есть общий сервер.

Если на этом шаге всё ок — связка рабочая, и я вставляю полный кокпит (все вкладки) в этот же проект.

## Запуск локально (необязательно)
```bash
npm install
cp .env.example .env   # и впиши свой anon ключ в .env
npm run dev
```

## Если что-то не работает
Открой на сайте консоль браузера (F12 → Console), скопируй красную ошибку и пришли мне — разберём. Частые причины: не добавлены переменные окружения в Vercel, не залита схема базы, или включено подтверждение email (тогда после регистрации надо подтвердить письмо).
